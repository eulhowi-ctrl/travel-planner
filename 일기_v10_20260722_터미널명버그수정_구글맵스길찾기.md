## 2️⃣ 버그 발견 — 시외버스터미널 명칭이 틀린 4개 도시

v9에서 이미 "정확한 업체명 없이는 `map.naver.com/p/search`가 조용히 일반 결과로 degrade된다"는 문제를 알고 있었기 때문에, `search.naver.com` 일반검색으로 우회해둔 상태였음. 이번 세션에서 실제 터미널명을 WebSearch로 하나씩 검증하다가 **더 근본적인 문제**를 발견:

> 강릉·경주·속초·전주는 "고속버스터미널"과 "시외버스터미널"이 서로 다른 위치의 별개 건물(운영사도 다름)인데, 서울에서 출발하는 노선은 그중 "고속버스터미널" 쪽으로만 감. 기존 코드는 무조건 `도시명 + 시외버스터미널`로 검색하고 있어서 이 4개 도시는 틀린 터미널을 가리키고 있었음.

| 도시 | 기존(틀림) | 수정 |
|---|---|---|
| 강릉 | 강릉시외버스터미널 | **강릉고속버스터미널** |
| 경주 | 경주시외버스터미널 | **경주고속버스터미널** (시외 쪽은 포항·울산·대구행) |
| 속초 | 속초시외버스터미널 | **속초고속버스터미널** |
| 전주 | 전주시외버스공용터미널 | **전주고속버스터미널** |
| 부산/여수/통영 | (원래도 단일 터미널이라 문제없음) | 부산종합/여수종합/통영종합버스터미널 |

Wikipedia·나무위키·교통 정보 사이트 검색으로 7개 도시 전부 정확한 명칭을 확인 후, `script.js`에 `BUS_TERMINAL_NAMES` 매핑을 추가하고 `naverBusTerminalSearchUrl()`이 이 매핑에 있으면 `map.naver.com/p/search/{정확한이름}`(POI 직결)을, 없으면 기존 `search.naver.com` 폴백을 쓰도록 수정.

## 3️⃣ 실제 "길찾기" 기능 시도 — 네이버 → 카카오 → 구글 순으로 검토

사용자가 "터미널을 기준으로" 실제 길찾기(턴바이턴) 버튼을 원함 ("각 터미널도 니가 알아서 찾아"). 세 가지 지도 서비스를 순서대로 검토:

1. **네이버지도**: 사용자가 붙여넣은 실제 `map.naver.com/p/directions/...` URL을 분석한 결과, 출발지·도착지 둘 다 네이버 내부 POI ID(2조각 코드)가 필요함을 재확인. 주소(ADDRESS_POI)조차 ID가 필요해서, 이름만으로는 생성 불가 — v9의 결론을 재확인만 하고 막힘 (naver.com은 WebFetch/브라우저 전면 차단, 재시도 안 함).
2. **카카오맵**: 사용자가 처음 선택한 방식(`map.kakao.com/link/to/...`)이지만, 이 URL 스킴은 위경도 좌표가 필수. 7개 터미널의 정확한 좌표를 WebSearch/WebFetch로 조사했으나:
   - 서울고속버스터미널만 위키백과에서 정확한 좌표 확보 (37.506008, 127.006987)
   - 나머지 6곳은 실패 — 나무위키는 403 차단(네이버처럼 취급됨), 위키백과엔 좌표 정보 자체가 없는 문서가 대부분, Google 지도 장소 페이지는 JS 렌더링이라 WebFetch로 못 읽음
   - 공공데이터포털에 전국 버스터미널 좌표 데이터셋이 있지만 API 키 발급이 필요해 이번 세션 범위 밖
   - → 좌표 조사가 막혔다는 사실을 사용자에게 보고하고 대안 제시
3. **구글맵스**: Directions URL API(`google.com/maps/dir/?api=1&origin=...&destination=...`)는 좌표 없이 **장소명 텍스트만으로** 서버사이드 지오코딩됨 — ID/좌표 조사가 아예 필요 없음. 사용자가 이 방식으로 전환 결정.

## 4️⃣ 구현 — `terminalDirectionsUrl()`

`script.js`에 추가:
```js
const SEOUL_TERMINAL_NAME = '서울고속버스터미널';
function terminalDirectionsUrl(cityName) {
    const exactName = BUS_TERMINAL_NAMES[cityName];
    if (!exactName) return null;
    return `https://www.google.com/maps/dir/?api=1&origin=${...}&destination=${...}&travelmode=driving`;
}
```
`BUS_TERMINAL_NAMES`에 없는 도시(예: 서울 자체)는 `null`을 반환해 버튼이 자동으로 숨겨짐. `search.html` 모달과 `itinerary.html` 교통 섹션 양쪽에 "🗺️ 길찾기" 버튼으로 노출.

**검증**: 로컬 정적 서버(`python -m http.server`) + Chrome 브라우저 자동화로 강릉 목적지 모달을 직접 열어 버튼 렌더링과 URL 구조(구글맵스 도메인, origin/destination 파라미터, 서울 엣지케이스에서 버튼 숨김) 확인.
