## 3️⃣ 해결된 주요 문제들
### Problem 1: API 키 없이 개발 진행 ❌ → ✅
**문제**: OpenWeather API 키 미발급 상태에서 날씨 기능 개발 필요
**해결**: `WEATHER_CONFIG.apiKey`가 비어있으면 `DEMO_WEATHER` 반환하는 **데모모드 패턴** 설계. 키 입력 시 코드 수정 없이 실제 API로 전환됨
### Problem 2: LLM API 없이 챗봇 구현 ❌ → ✅
**문제**: AI 챗봇 컨셉인데 백엔드 LLM 연동 불가
**해결**: 규칙 기반 인텐트 엔진으로 구현. 실배포 시 백엔드 LLM API로 교체 가능하도록 주석 표기
### Problem 3: 가격 듀얼 슬라이더 Min/Max 교차 ❌ → ✅
**문제**: 최소가격 슬라이더를 최대가격보다 오른쪽으로 끌면 값이 뒤집힘
**해결**: `handlePriceChange()`에서 교차 감지 시 자동 보정 로직 추가
### Problem 4: 쿠폰 조건 미검증 ❌ → ✅
**문제**: 5만원 쿠폰을 소액 예약에도 적용 가능했음
**해결**: `calculatePrice()`에서 10만원 미만이면 쿠폰 자동 해제 + 토스트 안내
### Problem 5: 페이지 간 데이터 단절 ❌ → ✅
**문제**: 검색→갤러리→위시리스트 이동 시 데이터 유실
**해결**: LocalStorage 키 5종으로 통일 (아래 4️⃣ 참조)

| `bookings` | 예약 내역 | booking |
| `selectedDestinationId` | 검색→갤러리 이동 시 전달 | search, gallery |
### 크로스 페이지 함수
```javascript
earnPoints(desc, points)   // rewards-script.js 정의, 외부 호출용
```
### 데모모드 패턴 (재사용 가능)
```javascript
const WEATHER_CONFIG = {
    apiKey: '',        // ← 키 넣으면 실제 API로 자동 전환
    demoMode: true
};
async function getWeather(city) {
    if (WEATHER_CONFIG.demoMode || !WEATHER_CONFIG.apiKey) {
        return DEMO_WEATHER[city] || DEMO_WEATHER['서울'];
    }
    // 실제 OpenWeather API 호출
}
```
