## 1️⃣ 세션 개요

"어디까지 했지"로 시작. v10 이후 상태 확인 후, 여수 숙소 확인 링크를 `search.naver.com` 일반검색에서 `hotels.naver.com` 실제 딥링크로 바꾸는 작업부터 시작해서, 세션이 진행되며 스코프가 계속 커짐:

1. 호텔 딥링크에 실제 체크인/체크아웃 날짜 반영 + 육로 목적지에도 날짜 선택 UI 추가
2. 여수 미식 여행 1일 코스에 실제 맛집/장소 큐레이션 (하드코딩)
3. 그 큐레이션이 시간 지나면 낡는다는 지적 → **카카오맵 JS SDK로 실시간 장소 검색** 도입 (여수 미식 여행 시범 적용 → 국내 전체로 일반화)
4. "국내 여행지가 너무 적다" → **국내 도시 100개 추가** (9개→109개, 생성 목적지 218개)
5. "코스 이동 시 교통편도 반영해줘" → 실시간 버스/지하철/택시 시간·요금 표시 시도. ODsay→Cloudflare Workers→Fly.io→Google Maps 순으로 각각 막혀서 최종적으로 **카카오맵 실제 길찾기 링크만** 제공하는 걸로 정착

## 2️⃣ 호텔 딥링크 실주소화 (`21c1611`, `2706175`)

- `naverHotelSearchUrl()`을 `search.naver.com` 일반검색에서 `hotels.naver.com/accommodation/search/{도시} 호텔/domestic?...` 실제 딥링크로 교체 (국내만, 해외는 URL 스킴 미확인이라 기존 폴백 유지)
- 사용자가 실제 hotels.naver.com에서 날짜 선택 후 나온 URL(`checkIn=`/`checkOut=` 파라미터 포함)을 제공해줘서, 그걸 기준으로 `getTripDates()` 값을 반영하도록 수정

- 브라우저로 실제 hotels.naver.com/flight.naver.com 새 탭까지 열어서 검증

## 3️⃣ 여수 미식 여행 실제 코스 큐레이션 (`f5f536f`)

기존 `buildGenericItinerary()`는 "OO 맛집 코스"/"식사 시간" 같은 플레이스홀더만 만들어서 실제 추천이 없었음. "네이버에서 검색 후 상위 3곳 중 정확도 가장 높은 곳" 요청 → naver.com 자동 스크래핑이 막혀있어 실시간 자동화는 불가하다고 설명하고, 대신 WebSearch로 직접 리서치해서 `HAND_WRITTEN_ITINERARIES['여수 미식 여행']`에 하드코딩:

교동시장(아침) → 로타리식당(게장백반) → 황제돌산갓김치(쇼핑) → 카페 라피끄(오션뷰) → 하멜등대 낭만포차거리(저녁)

각 장소는 다이닝코드/식신 등에서 평점·언급 빈도 비교 후 선정. 브라우저로 실제 렌더링 확인.

## 4️⃣ 카카오 로컬 라이브 검색 도입 (`ab9740b`, `7792f23`)

**문제 제기**: "니가 직접확인하면 10년 후 업데이트 내용이 반영이 안되잖아" — 정당한 지적. 검색엔진 대안 조사(구글/야후/다음/네이트) 결과 전부 CORS/API키/백엔드 문제로 실시간 자동화 불가 확인 후, **카카오맵 JS SDK**(REST API와 달리 스크립트 태그로 로드되어 CORS 문제 없음)로 방향 결정.

### 설정 과정에서 3단계 막힘 (전부 계정 설정 문제, 코드 문제 아님)
1. Kakao 앱의 "카카오맵" 제품이 기본 비활성화 상태 → `NotAuthorizedError: disabled OPEN_MAP_AND_LOCAL service` 에러로 확인, 사용자가 제품 설정에서 활성화
2. 도메인을 "제품 링크 관리"(카톡 공유 링크용)에 잘못 등록 → `AccessDeniedError: domain mismatched` 에러로 확인, "플랫폼 키" 메뉴가 맞는 위치였음
3. GitHub 시크릿이 빈 값으로 배포됨 → 배포된 파일 실제 내용(`jsKey: ''`)으로 확인, 재등록 후 해결

`kakao-places.js` 신규 (weather-api.js와 동일한 `__PLACEHOLDER__` 데모모드 패턴). `kakaoTopPlace(query)`가 관련도 1위 결과의 name/address/url(+나중에 lat/lng)을 반환. 여수 미식 여행 4개 항목에 `liveQuery` 필드 추가 → 실제 검색 결과로 자동 교체 확인(로타리식당→원조두꺼비게장 등).

이후 "다른 지역도 전부 여수처럼" 요청에 대해 손큐레이션은 627개 목적지에 불가능하다고 설명 → 같은 메커니즘을 `buildGenericItinerary()`에 일반화(한국만, 카카오 로컬이 국내 전용이라). 강릉 등 비-큐레이션 목적지에서도 실제 장소로 교체되는 것 확인.

## 5️⃣ 국내 목적지 100개 추가 (`4ead22f`)

기존 9개 도시(서울·부산·제주도·강릉·여수·경주·속초·전주·통영) → 100개 추가(광역시 6곳 포함 전국 전 권역) → 총 109개 도시, 생성 목적지 218개(기존 18개 + 신규 200개). 좌표는 일반 지리 지식 기반, 을왕리/성산일출봉 등 일부는 WebSearch로 검증.

**발견한 버그**: `isDomesticByLand` 체크가 `city !== '제주도'` 문자열 하나만 비교하고 있어서, 신규 추가한 제주 세부지역(성산/애월/우도/협재/서귀포)이 육로(KTX/버스)로 잘못 분류될 뻔함. `JEJU_CITIES` 배열로 일반화하고 `AIRPORT_CODES`에도 5곳 추가해서 항공권 딥링크가 CJU로 뜨도록 수정. 성산 항공권/안동·목포 KTX·버스/숙소 링크 전부 실클릭으로 검증.

## 6️⃣ 이동구간 실시간 교통편 — 3연속 막힘 끝에 링크로 정착 (`157dcd4` → `eace4b4` → `1d64527` → `ea1490c`)

"코스 내 A→B 이동 시 버스/택시/지하철 시간·요금 표기 + 대중교통 내비게이션 링크" 요청. 순서대로 시도하고 전부 사용자와 논의 후 결론:

| 시도 | 막힌 지점 | 결론 |
|---|---|---|
| ① ODsay(대중교통) + 카카오모빌리티(택시 추정) + Cloudflare Workers 프록시 | ODsay가 고정 아웃바운드 IP를 필수로 요구하는데 Cloudflare Workers는 Enterprise 전용 기능이라 불가 (`ApiKeyAuthFailed` 실제 사례로 확인) | Fly.io/VPS 등 고정IP 백엔드로 전환 논의하다 |
| ② (백엔드 자체를 보류) | "일단 링크형식으로 돌아가자" — 카카오맵 `/link/from/.../to/...` 실제 길찾기 딥링크로 대체 (URL 스킴은 브라우저로 직접 검증) | worker/ 디렉토리·워크플로 스텝 제거 |
| ③ "그래도 숫자까지 직접 표기" → Google Maps DirectionsService(클라이언트 SDK, 백엔드 불필요) | 한국 내 구글맵은 자동차/택시 길찾기 자체가 지원 안 됨(지도 반출 규제, 2026-02 완화 협의에도 여전), 대중교통도 부정확. 게다가 무료 크레딧 초과 시 자동 과금 | "돈나오는건 안되지" — 사용자가 리스크 거부, 최종 폐기 |
| **최종** | — | 코스 항목 사이에 "다음 장소로 이동 🗺️ 길찾기"로 카카오맵 실제 경로 링크만 표시. 숫자 없음, 과금 없음, 새 계정 불필요 |

전 과정에서 카카오맵 웹 UI 자동화(스크린샷/클릭)가 자주 렌더러 프리즈로 실패해서, 좌표 리서치는 부분적으로만 성공하고 결국 링크 방식으로 수렴한 것도 결정에 영향.

배포 사이트에서 최종 확인: 교동시장→원조두꺼비게장 사이 링크 클릭 시 실제 카카오맵 경로(`rt1=여수교동시장&rt2=원조두꺼비게장`) 정상 플롯.
